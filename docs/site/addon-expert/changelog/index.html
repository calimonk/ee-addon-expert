Changelog

Release history for Addon Expert. Full detail lives in CHANGELOG.md on GitHub; the highlights are below. Releases before 2.0.0 were published under the original fork name, Addon Manager +.

2.1.12026-06-17

Documentation and copy roll-up. No logic changes.

Changed

  • In-CP Documentation screen rewritten to cover the full feature set (was still describing only the ZIP uploader).
  • Fixed a stale "cached for 12h" label on the Releases screen — the interval has been 1 hour, with lazy refresh, since 1.4.0.
  • README refreshed with current screenshots; release package slimmed to runtime files only.

2.1.02026-06-17

Changed

  • Cleaner upload flow: a ZIP is inspected and scanned before committing. Incompatible packages are held on a confirm screen with the scan verdict and a one-click Force — no re-uploading.
  • Project repository renamed to calimonk/ee-addon-expert to match the product.

2.0.02026-06-17 — renamed to Addon Expert

Spun off into an independently-maintained product. Same codebase as 1.7.0 — an identity change plus an automatic config migration.

Changed

  • Short name addon_installer → addon_expert; product, namespace, and author updated. Javid Fazaeli credited as the original author of Addon Manager +.

Migration

  • On install, existing config (GitHub mappings, trust anchors, settings, overrides) and the release cache are copied from the old addon_installer to the new names — never clobbering, originals left intact.

1.7.0

Added

  • Compatibility feature scan: before you force-install an add-on that declares a newer PHP, the code is scanned for version-specific syntax/functions and you get an "appears safe" or "will fatal" verdict.

1.6.0

Added

  • Requirement override: force-install an add-on past an over-declared PHP/EE requirement, recorded with a persistent override badge and re-applied on future updates.
  • Packages screen flags not-yet-installed packages that EE would refuse as incompatible.

1.5.0

Added

  • Compatibility pre-flight check — reads the add-on's requires (PHP/EE) and refuses an incompatible install before extracting, surfacing the same verdict EE would, earlier.

1.4.x

Added

  • Auto-finalize: after an install, EE's update step runs automatically — for both GitHub installs and manual ZIP uploads.
  • Lazy parallel refresh of release info when the Releases screen loads; cache interval dropped to 1 hour.

Fixed

  • Inconsistent auto-finalize after ZIP uploads (opcache timing); audit-log formatting; banner version display.

1.3.x

Added

  • Supply-chain protection: trust-on-first-use repo identity pinning that blocks ownership transfer / RepoJacking, with a reconfirm path.
  • Append-only install Audit Log on its own screen.
  • CP Custom-menu integration with a configurable label and pending-update count.

Fixed

  • Self-update edge cases — backups moved outside the add-ons directory to avoid an autoloader collision; post-install redirect no longer 403s.

1.2.x

Added

  • GitHub release tracking and one-click install — the original feature that grew into Addon Expert.

Fixed

  • HTTP 415 on source-zipball downloads (Accept header).

Earlier 1.0–1.1 releases were the original Addon Manager + by Javid Fazaeli — the ZIP-upload installer this builds on.
