What it does

Install from ZIP

Upload an add-on package and it detects the folder containing addon.setup.php (even inside a wrapper), rejects unsafe paths, and extracts it into system/user/addons/.

Track GitHub releases

Map any add-on to a GitHub repo — author-declared in addon.setup.php or admin-mapped on the Releases screen. A daily-fresh check surfaces a GitHub: vX.Y.Z ↗ badge and a pending-update count in the sidebar.

One-click update & auto-finalize

Download, swap, and keep a backup of the previous version — then run EE's own update step automatically, so you never have to click through Developer → Add-Ons. Works for GitHub installs and manual uploads alike.

Supply-chain protection

On first install, the repo's stable GitHub identity is pinned. Every later install re-checks it and hard-blocks if it changed — catching ownership transfers and RepoJacking — with a "reconfirm" path for legitimate changes.

Compatibility checks & force

Refuses to install an add-on that needs a newer PHP/EE than your server runs. Choose to force it and Addon Expert scans the code first, telling you whether it's actually safe — "no PHP 8.3 features detected" vs "uses json_validate() — will fatal."

Audit log & CP menu

An append-only JSONL log records every install, block, trust pin, and override. An optional Custom-menu entry surfaces the pending-update count right in the sidebar.

Need to know

Requirements

ExpressionEngine 7, with the PHP ZipArchive extension (for uploads) and cURL (for GitHub downloads). GitHub API calls are unauthenticated — public repos only.
